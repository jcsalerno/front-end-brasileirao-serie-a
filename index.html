Documentação da API Brasileirão Série A 2025

Introdução

Esta API permite consultar informações sobre jogadores e times do Campeonato Brasileiro Série A 2025. Abaixo estão os endpoints disponíveis, como usá-los e exemplos de requisições e respostas.

URL Base: https://api-brasileirao-a.onrender.com/api

Observação: Os IDs dos times estão em ordem alfabética.

Atenção: Use esta API com moderação. No momento, nem todos os times estão disponíveis, e os tratamentos de erro ainda não foram implementados. Isso significa que:

  • Alguns times podem não retornar dados.
  • NENHUMA FOTO AINDA ESTÁ DISPONÍVEL
  • Requisições inválidas ou erros de servidor podem não ser tratados adequadamente.

Estamos trabalhando para melhorar a API e trazer mais funcionalidades em breve. Agradecemos a compreensão!

Campos Disponíveis

Você pode usar o parâmetro fields para filtrar os campos retornados nas respostas da API. Aqui estão todos os campos disponíveis:

Campos de Jogadores
  • id: ID do jogador.
  • nome: Nome do jogador.
  • numero: Número da camisa do jogador.
  • posicao: Posição do jogador (ex: Goleiro, Atacante).
  • aniversario: Data de nascimento do jogador (formato: dd/MM/yyyy).
  • cidade: Cidade de nascimento do jogador.
  • fotoUrl: URL da foto do jogador.
  • nacionalidade: Nacionalidade do jogador.
  • time: Informações sobre o time do jogador (objeto).
Campos de Times
  • id: ID do time.
  • nome: Nome do time.
  • cidadeSede: Cidade sede do time.
  • estadio: Nome do estádio do time.
  • anoFundacao: Ano de fundação do time.
  • jogadores: Lista de jogadores do time (array de objetos).

1. Buscar jogadores por time

Retorna uma lista de jogadores que pertencem a um time específico.

Endpoint
GET /api/jogadores/time/{timeId}
Parâmetros
  • timeId: O ID do time (ex: 7 para Flamengo).
  • fields (opcional): Lista de campos separados por vírgula para filtrar a resposta (ex: nome,numero,posicao).
Exemplo de Requisição
                        GET /api/jogadores/time/7?fields=nome,numero
Exemplo de Resposta

[
    {
        "nome": "Gabriel Delfim",
        "numero": 1
    },
    {
        "nome": "Everson",
        "numero": 22
    }
]

2. Buscar jogadores por posição

Retorna uma lista de jogadores que atuam em uma determinada posição.

Endpoint
GET /api/jogadores/posicao/{posicao}
Parâmetros
  • posicao: A posição dos jogadores (ex: "Goleiro", "Atacante").
  • fields (opcional): Lista de campos separados por vírgula para filtrar a resposta (ex: nome,numero).
Exemplo de Requisição
                        GET /api/jogadores/posicao/Goleiro?fields=nome,numero
Exemplo de Resposta

[
    {
        "nome": "Gabriel Delfim",
        "numero": 1
    },
    {
        "nome": "Everson",
        "numero": 22
    }
]

3. Listar todos os times

Retorna uma lista de todos os times cadastrados.

Endpoint
GET /api/times
Exemplo de Requisição
                        GET /api/times
Exemplo de Resposta

[
    {
        "id": 7,
        "nome": "Flamengo",
        "cidadeSede": "Rio de Janeiro",
        "estadio": "Maracanã",
        "anoFundacao": 1895
    },
    {
        "id": 8,
        "nome": "Fluminense",
        "cidadeSede": "Rio de Janeiro",
        "estadio": "Maracanã",
        "anoFundacao": 1902
    }
]

4. Buscar jogadores por nome do time

Retorna uma lista de jogadores que pertencem a um time específico, buscando pelo nome do time.

Endpoint
GET /api/times/{nome}/jogadores
Parâmetros
  • nome: O nome do time (ex: "Flamengo").
  • fields (opcional): Lista de campos separados por vírgula para filtrar a resposta (ex: nome,posicao).
Exemplo de Requisição
                        GET /api/times/Flamengo/jogadores?fields=nome,posicao
Exemplo de Resposta

[
    {
        "nome": "Gabriel Delfim",
        "posicao": "Goleiro"
    },
    {
        "nome": "Everson",
        "posicao": "Goleiro"
    }
]
